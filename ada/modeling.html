Advanced Data Modeling

Why This Matters

Imagine trying to build a skyscraper without a blueprint. You might stack bricks, but the moment you need to add a 50th floor, it collapses.

Data Modeling is that blueprint. It solves the Translation Problem: Business people speak in "Profits", computers speak in "Bits". A good model ensures they agree.

The Great Paradigm Wars

There are 4 main champions in data architecture. You must know these for the exam.

Paradigm Philosophy Key Concept
Inmon (CIF) Top-Down, 3NF, Enterprise View Single Source of Truth
Kimball Bottom-Up, Dimensional Star Schema (Facts & Dimensions)
Data Vault Hybrid, Auditable, Agile Hubs, Links, Satellites
Anchor (6NF) Highly Evolving, Temporal 6th Normal Form (Key-Value)

Slowly Changing Dimensions (SCD Types)

How do we handle change? (e.g., Someone moves from "London" to "Paris").

Type Description
Type 0 Fixed/Static. Never changes (e.g., Birth Date).
Type 1 Overwrite. No history. Updates the old value. (Typo correction).
Type 2 Add Row. The Industry Standard. Keep old row, insert new row with Effective Date.
Type 3 New Column. Add "Previous_City" column. Good for minimal history.
Type 4 History Table. Separate "Customer" (Current) and "Customer_History" tables.
Type 5 Type 4 + 1. Hybrid approach.
Type 6 Hybrid (1+2+3). Complex. Start with Type 2, overwrite a column (Type 1), keep history (Type 3).
EXAM QUESTION: What is SCD Type 2?
CORRECT ANSWER: Adding a new row with timestamps to track historical changes without deleting old data.

Data Vault Components

  • HUB (The Key): Contains only the unique Business Keys (e.g., Customer Hash Key, ID, Load Date). *Does not change.*
  • LINK (The Relationship): Connects Hubs together. (e.g., Customer bought Product).
  • SATELLITE (The Context): Where the actual data lives (Name, Address, Price). Can have multiple Satellites per Hub.
EXAM QUESTION: Which approach uses Hubs, Links, and Satellites?
CORRECT ANSWER: Data Vault.

Anchor Modeling (6NF)

An extreme form of normalization (6th Normal Form).

Instead of a table like `User(ID, Name, Email)`, you split it into 3 tables: `User_ID`, `User_Name(ID, Name)`, `User_Email(ID, Email)`.

Benefit: You can add a new attribute (e.g., `User_Twitter`) without touching the existing tables. Perfect for highly evolving schemas.

EXAM QUESTION: Which normal form is Anchor Modeling based on?
CORRECT ANSWER: 6th Normal Form (6NF).
