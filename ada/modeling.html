Advanced Data Modeling

Why This Matters

Imagine trying to build a skyscraper without a blueprint. You might stack bricks, but the moment you need to add a 50th floor or change the plumbing, the whole thing collapses.

Data Modeling is that blueprint for information. In the business world, it solves a massive problem: Translation. Business people speak in "profits" and "customers," while computers speak in "bits" and "tables." A good model ensures that when the CEO asks for "Total Sales," the database gives the correct number, not a random guess.

Foundations & Historical Context

Core Concept: The Schema

Jargon: "Schema"
Plain English: The designated "shape" of your data. Think of it like an empty Excel template with column headers (Name, Age, ID) before you type any data in.

Sibley’s Three-Level Architecture (1977)

Before we had modern clouds, Edgar Sibley defined the rules that still run the world. He split data systems into three layers to stop them from breaking:

  • 1. External Schema (The User View): What you see on your screen (e.g., an App interface).
  • 2. Conceptual Schema (The Logic): The maps and rules (e.g., "A User has one Password"). Independent of technology.
  • 3. Internal Schema (The Machine): How the hard drive strictly stores the 1s and 0s.

Frameworks: TOGAF vs. Zachman

You will hear these acronyms constantly. Here is the difference:

  • TOGAF (The Process): A step-by-step guide on how to build architecture.
    Analogy: The Recipe for baking a cake (Step 1, Step 2...).
  • Zachman (The Taxonomy): A categorization of what needs to be built.
    Analogy: The Ingredients List categorized by aisle (Dairy, Flour, Spices).

EXAM QUESTION: Which modeling technique primarily uses 6th Normal Form (6NF)?
CORRECT ANSWER: Anchor Modeling. It decomposes attributes into separate tables to handle high agility and temporal history.

Slowly Changing Dimensions (SCD)

Data changes over time. People move, products get renamed. "SCDs" are how we track this history.

The Top 3 Types (Midterm Essential)

  • Type 0 (Static): Never changes. (Example: Date of Birth).
  • Type 1 (Overwrite): The "Oops" fix. Replaces old data with new. No history kept. (Example: Correcting a typo in a name).
  • Type 2 (Add Row): The Gold Standard. When data changes, we keep the old row and add a new one with a new version number.
    Result: You have full history of exactly where a customer lived at any point in time.
EXAM QUESTION: What is the most common technique for preserving historical attribute values in a data warehouse?
CORRECT ANSWER: SCD Type 2 (Adding a new row with effective dates).

Deep Dive: 3rd Normal Form (3NF)

Inmon's preferred style. The goal is to eliminate redundancy.

The Mantra: "Every attribute must depend on the Key, the Whole Key, and Nothing but the Key."

  • 1NF: Atomic values (No lists in a single cell).
  • 2NF: No partial dependencies (Everything relates to the ID).
  • 3NF: No transitive dependencies (Columns shouldn't depend on other non-key columns).
