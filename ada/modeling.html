Core Warehousing Philosophies: Inmon vs. Kimball

The debate between the Inmon and Kimball approaches represents two fundamental schools of thought in data warehousing.

Feature Bill Inmon (CIF) Ralph Kimball (Dimensional)
Core Analogy Hub-and-spoke model. "Bus" architecture.
Design Approach Top-down: Build the central EDW first, then create data marts. Bottom-up: Build business-process-oriented data marts first.
Data Model Normalized (3NF) for the Enterprise Data Warehouse (EDW). Denormalized dimensional models (Star Schema) for data marts.
Single Source of Truth The EDW is the single source of truth. High data integrity. Achieved via "conformed dimensions" shared across marts.
Agility Less agile; requires significant upfront design. More agile; delivers business value faster.

Slowly Changing Dimensions (SCDs)

SCDs are techniques used to manage and track changes in dimensional data over time. This is critical for accurate historical reporting.

SCD Type 0: Retain Original

The original attribute value is kept. No changes are tracked. Used for attributes that are truly static or where changes are considered errors.

SCD Type 1: Overwrite

The old value is overwritten with the new value. No history is kept. Useful when correcting errors or when historical context is not important.

SCD Type 2: Add New Row

This is the most common and powerful type for historical analysis. When an attribute changes, a new row is added to the dimension table with the new value. The old row is marked as expired.


-- Example: Customer Dimension Table with SCD Type 2
| CustomerKey | CustomerID | Name      | City          | StartDate  | EndDate    | IsCurrent |
|-------------|------------|-----------|---------------|------------|------------|-----------|
| 101         | CUST-A1    | John Doe  | New York      | 2022-01-15 | 2023-05-20 | No        |
| 212         | CUST-A1    | John Doe  | San Francisco | 2023-05-21 | NULL       | Yes       |

SCD Type 3: Add New Attribute

A new column is added to the table to store the previous value of the attribute. This provides limited historical tracking (only the current and previous values).
