Data Mesh & Data Fabric

The 4 Pillars of Data Mesh

Zhamak Dehghani's paradigm shift from centralized monoliths to decentralized socio-technical architecture.

  1. Domain-Oriented Ownership: Decentralize data ownership to the people who know it best.
  2. Data as a Product: Apply product thinking to data. It must be discoverable, addressable, trustworthy, self-describing, interoperable, and secure.
  3. Self-Serve Data Infrastructure: A platform that abstracts complexity, allowing domains to build valid data products easily.
  4. Federated Computational Governance: Global standards applied locally to ensure interoperability.

Data Fabric: The Metadata Engine

While Data Mesh is organizational, Data Fabric is technological. It uses active metadata, knowledge graphs, and AI to weave together disparate data sources.

  • Auto-Discovery: Automatically inferring schemas and lineage.
  • Augmented Data Management: AI-driven recommendations for integration.
  • Virtualization: Accessing data where it lives without moving it.
